# APM Marketplace Workflow

## Overview

Gem Team uses APM (Agent Package Manager) as the primary distribution method. This document explains how the release workflow integrates with APM.

## File Structure

```
gem-team/
├── apm.yml                              # APM manifest (source of truth)
├── plugin.json                          # Plugin manifest (for compatibility)
├── .claude-plugin/
│   └── marketplace.json                 # Auto-generated by apm pack
├── .apm/
│   └── agents/                          # Agent definitions
├── release-please-config.json           # Release automation config
└── .release-please-manifest.json        # Current version
```

## Release Workflow

### 1. Development

```bash
# Make changes to agents, update docs, etc.
git add .
git commit -m "feat: add new agent"
git push
```

### 2. Automated Release (via release-please)

When you push to `main`, release-please:

1. Creates a PR with version bumps in:
   - `apm.yml`
   - `plugin.json`
   - `.claude-plugin/marketplace.json`
   - `version.txt`
   - `README.md`
   - `CHANGELOG.md`

2. When PR is merged, creates:
   - Git tag: `gem-team-v1.X.X`
   - GitHub release

### 3. Manual Build (for testing)

```bash
# Build the plugin bundle and marketplace
apm pack

# Validate marketplace configuration
apm marketplace check

# Output:
# - build/gem-team-1.X.X/          (plugin bundle)
# - .claude-plugin/marketplace.json (marketplace manifest)
```

## Installation Methods

### For End Users

**Recommended (APM):**

```bash
apm install mubaidr/gem-team
```

**Via Marketplace:**

```bash
apm marketplace add mubaidr/gem-team
apm install gem-team@gem-team
```

### For Tool-Specific Users

All tools use APM:

```bash
# Works for Claude Code, Cursor, OpenCode, GitHub Copilot CLI
apm marketplace add mubaidr/gem-team
apm install gem-team@gem-team
```

APM automatically detects the tool and installs to the correct location.

## Key Configuration

### apm.yml

```yaml
marketplace:
  owner:
    name: mubaidr
    url: https://github.com/mubaidr
    email: mubaidr@gmail.com

  build:
    tagPattern: "gem-team-v{version}" # Matches release-please tags

  packages:
    - name: gem-team
      source: mubaidr/gem-team
      version: "^1.19.0"
```

### release-please-config.json

```json
{
  "extra-files": [".claude-plugin/marketplace.json", "apm.yml", "plugin.json", "README.md", "version.txt"]
}
```

## Verification

```bash
# Check everything is configured correctly
apm marketplace check

# Expected output:
# [+] All 1 entries OK
```

## Troubleshooting

### "No tag matching version"

- Ensure git tags follow pattern: `gem-team-v1.X.X`
- Check `build.tagPattern` in `apm.yml`

### "plugin.json not found" warning

- This is informational only
- APM synthesizes from `apm.yml` if missing
- We include explicit `plugin.json` for clarity

### Marketplace validation fails

- Run `apm marketplace check` for details
- Verify git tags are pushed to GitHub
- Check network connectivity

## Benefits

✅ **Single source of truth** - `apm.yml` drives everything
✅ **Automated versioning** - release-please handles all version bumps
✅ **Multi-tool support** - Works with Claude Code, Cursor, OpenCode, GitHub Copilot CLI
✅ **Version locking** - Resolves to specific git SHAs
✅ **Validation** - `apm marketplace check` ensures correctness
