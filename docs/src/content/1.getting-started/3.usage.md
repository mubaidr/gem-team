## Basic Workflow

Gem Team adapts its workflow depth to task complexity. Here's how to use it day-to-day:

### 1. Start a Task

Simply describe what you need. Gem Team's Orchestrator will:

1. **Phase 0 — Init & Clarify**: Read context, detect intent, classify complexity.
2. **Phase 1 — Route**: Route to the right workflow path.
3. **Phase 2 — Plan**: Create an execution plan matched to complexity.
4. **Phase 3 — Execute**: Delegate to specialist agents in waves with integration gates.
5. **Phase 4 — Output**: Present results with status and progress.

### 2. Understanding Complexity Routing

| Complexity  | When                                                     | What Happens                                                       |
| ----------- | -------------------------------------------------------- | ------------------------------------------------------------------ |
| **TRIVIAL** | Obvious mechanical task (typo fix, renames)              | Direct execution with a tiny checklist                             |
| **LOW**     | Small bounded task (1-2 files, known pattern)            | Lightweight in-memory planning and execution                       |
| **MEDIUM**  | Multiple files, new patterns, moderate uncertainty       | Durable plan, context envelope, wave execution, integration review |
| **HIGH**    | Architecture changes, API/schema/auth, broad regressions | Full planner, reviewer, critic; every wave gated                   |

### 3. Working with Plans

For MEDIUM and HIGH complexity tasks, Gem Team creates durable plan artifacts:

```
docs/plan/{plan_id}/
  plan.yaml              # DAG-based task breakdown
  context_envelope.json  # Progressive context cache
  tasks/                 # Individual task results
```

Plans are **resumable** — you can pause, inspect, and continue cleanly.

### 4. Learning & Persistence

After each task, Gem Team persists high-confidence learnings:

- **Product decisions** → PRD (`docs/PRD.yaml`)
- **Technical conventions** → `AGENTS.md`
- **Patterns & gotchas** → Memory (repo/session/global)
- **Repeatable workflows** → Skills (`docs/skills/`)

## Tips

- **Be specific** in your requests — clearer input leads to better plans.
- **Review plan artifacts** for MEDIUM/HIGH tasks before execution.
- **Use plan IDs** to reference and resume ongoing work, e.g., `continue plan 20250601-auth-refactor`.
- **Keep `apm_modules/` out of git** — it's an install cache.
