Cost-consciousness and performance are core design principles. This page covers optimization across agent execution, context management, and build pipeline.

## Agent Execution Optimization

> **Model routing recommendations** are on the [Agents page →](/agents). Below covers output hygiene and researcher budgets.

### Output Hygiene

Agents are configured to minimize token waste:

- **Native tool flags**: Agents use `--quiet`, `--oneline`, `maxResults` to truncate output at source
- **Pipeline truncation**: `head`/`tail`/`grep` used only when native flags are insufficient
- **Context pruning**: The context envelope is filtered per-agent — each agent receives only the sections relevant to its task

### Researcher Budget Controls

The `gem-researcher` agent has configurable budgets to prevent runaway exploration:

```yaml
orchestrator:
  researcher:
    max_searches: 10 # Max search calls per task
    max_files_to_read: 25 # Max files read per task
    max_depth: 3 # Max exploration depth (follow-up searches)
```

## Context & Cache Optimization

> Context envelope structure and memory tiers are covered in [Core Concepts →](/guide/core-concepts). This section covers eviction and efficiency settings.

The context envelope is filtered per-agent (each receives only relevant sections). Facts are persisted only at confidence ≥ 0.85, promoted to durable cache at ≥ 0.90 after ≥ 3 uses. Auto-eviction at 90 days of no access.

## Build & CI/CD Optimization

### Dependency Caching

The CI workflow caches `node_modules` via `actions/setup-node` with `cache: npm`:

```yaml
- uses: actions/setup-node@v4
  with:
    node-version: "22"
    cache: npm
    cache-dependency-path: docs/src/package-lock.json
```

### Prerendering Strategy

The docs site uses static site generation (SSG) with prerendering for optimal CDN delivery:

```ts [nuxt.config.ts]
nitro: {
  prerender: {
    routes: ["/"],
    crawlLinks: true,
  },
}
```

This ensures:

- HTML is pre-built at deployment time
- No server-side rendering overhead at request time
- Maximum cacheability on CDN edge nodes

## Bundle Size Optimization

### Tree-shaking & Auto-imports

Nuxt's auto-import system tree-shakes unused exports, ensuring only used components and composables are included in the final bundle.

### Image Optimization

[Nuxt Image](https://image.nuxt.com/) is configured to:

- Automatically convert images to WebP/AVIF formats
- Generate responsive sizes
- Support lazy loading with `loading="lazy"` and `fetchpriority="low"`
- Use `preload` with `fetchPriority: "high"` for critical LCP images

```html
<NuxtImg
  src="/hero-banner.jpg"
  format="webp"
  :preload="{ fetchPriority: 'high' }"
  loading="eager"
  width="200"
  height="100"
/>
```

### Lazy Hydration

Components that don't need immediate interactivity can defer hydration:

```html
<LazyMyComponent hydrate-on-visible />
```

This improves Time-to-Interactive (TTI) by deferring JavaScript execution for below-the-fold content.

## Measuring & Monitoring

### Built-in Tools

- **Nuxt DevTools**: Timeline, Assets, Render Tree, and Inspect panels for performance analysis
- **`nuxi analyze`**: Visualize bundle composition to identify large dependencies
- **Chrome DevTools**: Performance panel for LCP/CLS/INP metrics; Lighthouse for comprehensive audit

### Continuous Monitoring

Enable the link checker to catch broken links that degrade SEO and user experience:

```ts [nuxt.config.ts]
modules: [
  "@nuxtjs/seo",
],
```

> The built-in SEO module includes link checking, sitemap generation, and robots.txt management.
