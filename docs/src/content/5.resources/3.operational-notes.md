## Best Practices

### Project-Scoped Installs

Prefer project-scoped installs for teams so `apm.yml` and `apm.lock.yaml` make the setup reproducible across environments and CI.

### Version Pinning

Pin releases with `#vX.Y.Z` for stable CI and team onboarding:

```bash
apm install mubaidr/gem-team#v1.20.0 --target copilot
```

### APM Modules

Keep `apm_modules/` out of git — it is an install cache. Add it to your `.gitignore`:

```
apm_modules/
```

### Auditing

- Run `apm audit` before release and in CI to verify package integrity.
- Review generated files before committing large updates.

### Approval-Gated Operations

Treat the following as approval-gated tasks:

- DevOps production deployment
- Infrastructure provisioning
- Data migration
- Destructive operations

These should always require explicit approval before execution.

### AGENTS.md vs. Plan Context

- Keep **project rules** in `AGENTS.md` — these are stable, cross-cutting conventions.
- Keep **task-specific context** in `docs/plan/{plan_id}/` — these are scoped to individual plan executions.
